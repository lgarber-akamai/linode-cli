Images
======

This section details Image-related Linode CLI commands.

.. _images_create:

`create <https://techdocs.akamai.com/linode-api/reference/post-image>`_
-----------------------------------------------------------------------


Captures a private, gold-master image from a Linode disk.

.. cssclass:: action-subheading

    .. rubric:: Usage

:action-subheading-description:`The format accepted by this command.`

..  code-block:: bash

    linode-cli images create [-h] [--disk_id disk_id]
    	[--cloud_init cloud_init] [--label label]
    	[--description description] [--tags tags]


.. cssclass:: action-subheading

    .. rubric:: Sample

:action-subheading-description:`Examples of how this command might be used.`


..  code-block:: bash

    linode-cli images create \
    	--label this_is_a_label \
    	--description "A longer description \
    		of the image" \
    	--disk_id 123


.. cssclass:: action-subheading

    .. rubric:: Arguments

:action-subheading-description:`Additional fields used to execute this request.`

..  list-table::
    :header-rows: 1
    :width: 100%
    :widths: 15 10 10 65

    * - Name
      - Required
      - Type
      - Description

    * - :code:`--disk_id`
      - :action-argument-required:`Yes`
      - integer
      - The ID of the Linode Disk that this image will be created from.
    * - :code:`--cloud_init`
      - No
      - boolean
      - Whether this image supports `cloud-init <https://www.linode.com/docs/guides/write-files-with-cloud-init/>`_.
    * - :code:`--description`
      - No
      - string
      - A detailed description of this image.
    * - :code:`--label`
      - No
      - string
      - A short title of this image.
    * - :code:`--tags`
      - No
      - array[string]
      - Tags used for organizational purposes.

.. _images_list:

`list <https://techdocs.akamai.com/linode-api/reference/get-images>`_
---------------------------------------------------------------------


Returns a paginated list of images.

.. cssclass:: action-subheading

    .. rubric:: Usage

:action-subheading-description:`The format accepted by this command.`

..  code-block:: bash

    linode-cli images list [-h] [--is_public is_public]
    	[--deprecated deprecated] [--size size] [--tags [tags] ]
    	[--status status] [--vendor vendor] [--label label]
    	[--type type]
    	[--order-by {is_public,deprecated,size,tags,status,vendor,label,type}]
    	[--order {asc,desc}]


.. cssclass:: action-subheading

    .. rubric:: Sample

:action-subheading-description:`Examples of how this command might be used.`


..  code-block:: bash

    linode-cli images list


.. rubric:: Filterable Attributes

`Arguments used to define a filter for response entries.`

..  list-table::
    :header-rows: 1
    :width: 100%
    :widths: 15 10 75

    * - Name
      - Type
      - Description

    * - :code:`--deprecated`
      - *boolean*
      - Whether this image is deprecated.
    * - :code:`--is_public`
      - *boolean*
      - Revealed as :code:`true` if the image is a public distribution image.
    * - :code:`--label`
      - *string*
      - A short description of the image.
    * - :code:`--size`
      - *integer*
      - The minimum size in MB this image needs to deploy.
    * - :code:`--status`
      - *string*
      - The current status of the image.
    * - :code:`--tags`
      - *array[string]*
      - Tags used for organizational purposes.
    * - :code:`--type`
      - *string*
      - How the image was created.
    * - :code:`--vendor`
      - *string*
      - The upstream distribution vendor.

.. _images_upload:

`upload <https://techdocs.akamai.com/linode-api/reference/post-upload-image>`_
------------------------------------------------------------------------------


Creates a new private image instance and returns a URL as the :code:`upload_to` object in the response.

.. cssclass:: action-subheading

    .. rubric:: Usage

:action-subheading-description:`The format accepted by this command.`

..  code-block:: bash

    linode-cli images upload [-h] [--region region]
    	[--cloud_init cloud_init] [--tags tags]
    	[--description description] [--label label]


.. cssclass:: action-subheading

    .. rubric:: Sample

:action-subheading-description:`Examples of how this command might be used.`


..  code-block:: bash

    # Upload the image file in a single step
    linode-cli image-upload \
    	--description "Optional details about the Image" \
    	--label "Example Image" \
    	--region us-east \
    	/path/to/image-file.img.gz

    # Returns the upload_to URL
    linode-cli images upload \
    	--description "Optional details about the Image" \
    	--label "Example Image" \
    	--region us-east


.. cssclass:: action-subheading

    .. rubric:: Arguments

:action-subheading-description:`Additional fields used to execute this request.`

..  list-table::
    :header-rows: 1
    :width: 100%
    :widths: 15 10 10 65

    * - Name
      - Required
      - Type
      - Description

    * - :code:`--label`
      - :action-argument-required:`Yes`
      - string
      - Label for the uploaded image.
    * - :code:`--region`
      - :action-argument-required:`Yes`
      - string
      - The region to upload to.
    * - :code:`--cloud_init`
      - No
      - boolean
      - Whether the uploaded Image supports cloud-init.
    * - :code:`--description`
      - No
      - string
      - Description for the uploaded image.
    * - :code:`--tags`
      - No
      - array[string]
      - Tags you can use to organize images.

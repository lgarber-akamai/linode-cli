Domains

This section details Domain-related Linode CLI commands.

.. _domains_clone:

`clone <https://techdocs.akamai.com/linode-api/reference/post-clone-domain>`_
-----------------------------------------------------------------------------


Clones a Domain and all associated DNS records from a Domain that is registered in Linode's DNS manager.

.. cssclass:: action-subheading

    .. rubric:: Usage

:action-subheading-description:`The format accepted by this command.`

..  code-block:: bash

    linode-cli domains clone [-h] [--domain domain] domainId


.. cssclass:: action-subheading

    .. rubric:: Sample

:action-subheading-description:`Examples of how this command might be used.`


..  code-block:: bash

    linode-cli domains clone 123 --domain example.com

.. _domains_create:

`create <https://techdocs.akamai.com/linode-api/reference/post-domain>`_
------------------------------------------------------------------------


Adds a new Domain to Linode's DNS Manager.

.. cssclass:: action-subheading

    .. rubric:: Usage

:action-subheading-description:`The format accepted by this command.`

..  code-block:: bash

    linode-cli domains create [-h] [--master_ips master_ips]
    	[--ttl_sec ttl_sec] [--retry_sec retry_sec]
    	[--axfr_ips axfr_ips] [--soa_email soa_email]
    	[--description description] [--type type]
    	[--refresh_sec refresh_sec] [--tags tags] [--status status]
    	[--domain domain] [--group group] [--expire_sec expire_sec]


.. cssclass:: action-subheading

    .. rubric:: Sample

:action-subheading-description:`Examples of how this command might be used.`


..  code-block:: bash

    linode-cli domains create \
    	--type master \
    	--domain example.org \
    	--soa_email admin@example.org


.. cssclass:: action-subheading

    .. rubric:: Arguments

:action-subheading-description:`Additional fields used to execute this request.`

..  list-table::
    :header-rows: 1
    :width: 100%
    :widths: 15 10 10 65

    * - Name
      - Required
      - Type
      - Description

    * - :code:`--domain`
      - :action-argument-required:`Yes`
      - string
      - The domain this Domain represents.
    * - :code:`--type`
      - :action-argument-required:`Yes`
      - string
      - Whether this Domain represents the authoritative source of information for the domain it describes (:code:`master`), or whether it is a read-only copy of a master (:code:`slave`).
    * - :code:`--axfr_ips`
      - No
      - array[string]
      - The list of IPs that may perform a zone transfer for this Domain.
    * - :code:`--description`
      - No
      - string
      - A description for this Domain.
    * - :code:`--expire_sec`
      - No
      - integer
      - The amount of time in seconds that may pass before this Domain is no longer authoritative.
    * - :code:`--group`
      - No
      - string
      - The group this Domain belongs to.
    * - :code:`--id`
      - No
      - integer
      - This Domain's unique ID.
    * - :code:`--master_ips`
      - No
      - array[string]
      - The IP addresses representing the master DNS for this Domain.
    * - :code:`--refresh_sec`
      - No
      - integer
      - The amount of time in seconds before this Domain should be refreshed.
    * - :code:`--retry_sec`
      - No
      - integer
      - The interval, in seconds, at which a failed refresh should be retried.
    * - :code:`--soa_email`
      - No
      - string
      - Start of Authority email address.
    * - :code:`--status`
      - No
      - string
      - Used to control whether this Domain is currently being rendered.
    * - :code:`--tags`
      - No
      - array[string]
      - An array of tags applied to this object.
    * - :code:`--ttl_sec`
      - No
      - integer
      - "Time to Live" - the amount of time in seconds that this Domain's records may be cached by resolvers or other domain servers.
